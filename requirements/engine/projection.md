# Projection

| Path | Tag | Requirement |
|------|-----|-------------|
| R:engine / projection / net-worth | CORE | The plan must produce a net worth projection — a stream spanning the full [D:streams / timeline](../conceptual-model.md#d-streams) that yields the household's total net financial position for every year. |
| R:engine / projection / net-worth : composition | CORE | The net worth value for each year must equal the sum of all [S:assets](../conceptual-model.md#s-assets) stream values minus the sum of all [S:liabilities](../conceptual-model.md#s-liabilities) stream values for that year. |
| R:engine / projection / net-worth / assets | CORE | The net worth projection must decompose into constituent [S:assets](../conceptual-model.md#s-assets) streams — accounts, hard assets, and business entities — each contributing its projected year-value independently. |
| R:engine / projection / net-worth / assets : value-grounding | CORE | Every asset stream must produce a projected value for each year in [D:streams / timeline](../conceptual-model.md#d-streams). For financial accounts, that value is the projected account balance. For hard assets and businesses, that value is the projected market value, gross of any liabilities against the asset. |
| R:engine / projection / net-worth / assets / retirement | MVP | The plan must produce a retirement aggregate stream whose year-value is the sum of all [S:assets / accounts / [ member ] / [ 401k ]](../conceptual-model.md#s-assets), [S:assets / accounts / [ member ] / [ roth-401k ]](../conceptual-model.md#s-assets), [S:assets / accounts / [ member ] / [ ira ]](../conceptual-model.md#s-assets), and [S:assets / accounts / [ member ] / [ roth-ira ]](../conceptual-model.md#s-assets) balances for that year. |
| R:engine / projection / net-worth / assets / health | MVP | The plan must produce a health aggregate stream whose year-value is the sum of all [S:assets / accounts / [ member ] / [ hsa ]](../conceptual-model.md#s-assets) balances for that year. |
| R:engine / projection / net-worth / assets / taxable | MVP | The plan must produce a taxable aggregate stream whose year-value is the sum of all [S:assets / accounts / [ member ] / [ bank ]](../conceptual-model.md#s-assets), [S:assets / accounts / [ member ] / [ brokerage ]](../conceptual-model.md#s-assets), [S:assets / accounts / [ bank ]](../conceptual-model.md#s-assets), and [S:assets / accounts / [ brokerage ]](../conceptual-model.md#s-assets) balances for that year. |
| R:engine / projection / net-worth / assets / hard-assets | MVP | The plan must produce a hard-assets aggregate stream whose year-value is the sum of all [S:assets / hard-assets](../conceptual-model.md#s-assets) stream values and [S:assets / [ business ]](../conceptual-model.md#s-assets) values for that year. |
| R:engine / projection / net-worth / assets / [ business ] | FUT | Each [S:assets / [ business ]](../conceptual-model.md#s-assets) must be a named aggregate stream whose year-value is the sum of its constituent child valuation streams — including the intrinsic value of the business itself (goodwill, enterprise value) and the projected value of any assets held by the business. |
| R:engine / projection / net-worth / liabilities | CORE | The net worth projection must decompose into constituent [S:liabilities](../conceptual-model.md#s-liabilities) streams — amortized loans, interest-only obligations, and revolving credit — each contributing its outstanding balance independently. |
| R:engine / projection / net-worth / liabilities : value-grounding | CORE | Every liability stream must produce a projected value for each year in [D:streams / timeline](../conceptual-model.md#d-streams). That value is the remaining balance owed — the outstanding principal for amortized and interest-only loans, and the outstanding balance for revolving credit. |
| R:engine / projection / net-worth / liabilities / long-term | MVP | The plan must produce a long-term liabilities aggregate stream whose year-value is the sum of all [S:liabilities / [ amortized-loan ]](../conceptual-model.md#s-liabilities) and [S:liabilities / [ interest-only-loan ]](../conceptual-model.md#s-liabilities) balances for that year. |
| R:engine / projection / net-worth / liabilities / short-term | MVP | The plan must produce a short-term liabilities aggregate stream whose year-value is the sum of all [S:liabilities / [ credit-line ]](../conceptual-model.md#s-liabilities) balances for that year. |
| R:engine / projection / net-worth : household-lifecycle | CORE | The net worth projection must reflect the effect of [S:household / [ member ]](../conceptual-model.md#s-household) lifecycle events — including retirement, death, and dependent phase transitions — on asset and liability streams for each year they occur. |
| R:engine / projection / account-balance | CORE | Each [S:assets / accounts](../conceptual-model.md#s-assets) stream must produce a projected year-end balance for every year in [D:streams / timeline](../conceptual-model.md#d-streams). |
| R:engine / projection / account-balance / recurrence : end-of-year | MVP | [S:assets / accounts](../conceptual-model.md#s-assets) balances must be projected using [D:formulas / balance-recurrence / end-of-year](../conceptual-model.md#d-formulas). |
| R:engine / projection / account-balance / recurrence : beginning-of-year | FUT | [S:assets / accounts](../conceptual-model.md#s-assets) balances must support [D:formulas / balance-recurrence / beginning-of-year](../conceptual-model.md#d-formulas). |
| R:engine / projection / account-balance / recurrence : mid-year | FUT | [S:assets / accounts](../conceptual-model.md#s-assets) balances must support [D:formulas / balance-recurrence / mid-year](../conceptual-model.md#d-formulas). |
| R:engine / projection / account-balance : rate-derivation | MVP | The effective growth rate applied in [D:formulas / balance-recurrence](../conceptual-model.md#d-formulas) must be the weighted average of [S:assumptions / rates](../conceptual-model.md#s-assumptions) using [S:assumptions / allocations](../conceptual-model.md#s-assumptions) as weights. |
| R:engine / projection / account-balance : floor | CORE | No [S:assets / accounts](../conceptual-model.md#s-assets) balance may go below zero. |
