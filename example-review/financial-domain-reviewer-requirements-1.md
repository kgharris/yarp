# Review: Financial Domain Reviewer
**Phase:** requirements | **Iteration:** 1 | **Date:** 2026-04-08

## Summary

The requirements artifacts define a solid core of retirement planning mechanics for MVP scope, but contain several financially significant gaps and inaccuracies that would produce incorrect projections if built as-is. The most critical issues are: the Social Security claiming age constraint incorrectly ties the valid range to FRA instead of the statutory 62-70 window; the RMD start-age mapping omits the SECURE 2.0 three-tier structure (73/75 by birth year); Roth IRA withdrawals lack the five-year rule and ordering rules that determine taxability; and health insurance phase logic is per-household when it must be per-member since spouses can reach Medicare eligibility at different ages. The conceptual model's policy parameter catalog is thorough and well-structured, but the engine requirements that consume those parameters are often underspecified on the financial rules that govern how they interact.

## Passed Checks

- **Denomination framework**: The four-frame denomination model (YZV/CYV/FYV/PYV) is financially sound and correctly distinguishes stored values, display values, projection output, and historical actuals. The requirement that all storage is YZV is appropriate for a planning tool.
- **Balance recurrence formulas**: The three recurrence variants (end-of-year, beginning-of-year, mid-year) are correctly formulated. The end-of-year convention as MVP default is a reasonable simplification.
- **CPI-W for Social Security COLA**: Correctly identified as the appropriate index for SS COLA adjustments, distinct from CPI. The MVP simplification of defaulting CPI-W to CPI is acceptable and documented.
- **Healthcare inflation separation**: Correctly modeled as a distinct inflation rate from CPI, reflecting the historical divergence of healthcare costs from general inflation.
- **RMD as withdrawal floor**: Correctly specified that RMD is a minimum, not a fixed amount — voluntary withdrawals above the RMD are permitted.
- **401k annual additions limit**: Correctly includes employer contributions in the overall cap, per IRC Section 415(c).
- **Roth IRA phase-out**: Correctly specified as MAGI-based with the phase-out concept from the conceptual model.
- **Standard deduction inflation adjustment**: Correctly specified as CPI-indexed with per-year override capability.
- **Medicare eligibility age**: Correctly modeled as configurable (currently 65, but legislation could change).
- **Dependent age-out**: Correctly modeled as a configurable parameter that affects household budget calculations.
- **Account balance floor at zero**: Correctly prevents negative balances, which is a real-world constraint.
- **SS taxation thresholds not inflation-indexed**: The conceptual model correctly notes these thresholds are fixed by statute and not adjusted for inflation, which is accurate per current law (the thresholds have not changed since 1993).
