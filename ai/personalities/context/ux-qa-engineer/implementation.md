# UX QA Engineer — Implementation Phase

## Focus

- Do cues update correctly across all views when context changes — denomination switch, assumption change, scenario switch — including tooltips, secondary panels, and cached values?
- Does progressive disclosure maintain correct state through navigation — collapse, expand, leave, return?
- Does the built behavior match the specified behavior for edge cases, not just the happy path?
- Are error messages specific enough for the user to know what went wrong and what to do?
- Do state transitions behave correctly when the user acts faster than the system responds?

## Artifacts

Read frontend source and the running application exhaustively.
