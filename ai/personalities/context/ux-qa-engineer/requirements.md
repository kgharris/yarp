# UX QA Engineer — Requirements Phase

## Focus

- What scenarios would produce missing or misleading cues — denomination changes mid-workflow, stale values after assumption updates, ambiguous labels?
- What interactions could break the user's mental model — navigating back after state changes, deep links without context, losing track of which scenario is active?
- Can progressive disclosure fail the user — hidden detail they need, defaults inappropriate for their situation, complexity surfacing unexpectedly?
- Are success and failure paths both specified? Are there interaction sequences nobody thought to address?
- Are there implicit orderings a user might violate?
