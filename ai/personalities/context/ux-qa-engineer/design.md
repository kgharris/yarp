# UX QA Engineer — Design Phase

## Focus

- Does the design degrade gracefully — do cues, navigation context, and progressive disclosure state survive error conditions?
- Can denomination switching produce inconsistent displays across views, tooltips, or cached panels?
- Does the user know what's stale and what's current when a computation fails or data is loading?
- Are destructive actions protected by confirmation?
- Can the user end up in a state where progressive disclosure hides information they need to understand an error or make a decision?
