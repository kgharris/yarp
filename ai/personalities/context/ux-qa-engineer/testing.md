# UX QA Engineer — Testing Phase

## Focus

- Do test cases cover cue consistency across context changes — denomination switching, scenario changes, assumption updates verified across multiple views?
- Do tests cover progressive disclosure scenarios — expand, collapse, navigate away, return, verify state preserved?
- Do test cases cover the interaction sequences most likely to break (rapid input, mode switching, interrupted workflows)?
- Do they cover state transitions, not just static renders?
- Are boundary inputs (empty, maximum, zero) represented?
