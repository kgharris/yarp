# Database Engineer — Design Phase

## Focus

- Is the data model correct, complete, easy to use, and maintainable?
- Does the schema structure support the persistence requirements without redundancy?
- Are field names, types, and naming conventions consistent?
- Is the migration approach defined for schema evolution?
- Does the storage format design satisfy durability and atomicity guarantees?
- Are import/export contracts defined?
- When and where does validation run in the data lifecycle?
- Are architectural boundaries from the principles chain respected?
