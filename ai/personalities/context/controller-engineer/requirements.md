# Controller Engineer — Requirements Phase

## Focus

- Are application workflows specified end-to-end (user action → computation → result presentation)?
- Are state transitions clear (when does data change, when is it persisted)?
- Do requirements specify how updates propagate (when assumptions change, what views update, when user sees results)?
- Do requirements imply realistic interactions between core computational functions and user-facing views?
- Is the flow of data through the system unambiguous?
