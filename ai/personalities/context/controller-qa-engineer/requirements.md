# Controller QA Engineer — Requirements Phase

## Focus

- Are acceptance criteria stated clearly enough to verify (what constitutes success/failure)?
- Are error conditions and edge cases specified for workflows?
- Are the interactions between computational functions and user views verifiable?
- Can requirements be tested end-to-end (clear inputs, expected outputs, observable results)?
