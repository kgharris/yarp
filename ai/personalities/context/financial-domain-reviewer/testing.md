# Financial Domain Reviewer — Testing Phase

## Focus

- Are test cases using realistic scenarios that would expose incorrect rule application — not just mathematically consistent but financially wrong answers?
- Do test scenarios cover the financial distinctions the law requires — birth-year cohort boundaries, account type differences (Roth vs. traditional, self-only vs. family HSA), filing status variations?
- Are there financial rules modeled in the system with no test scenario exercising them?

## Artifacts

Read test scenarios and test fixture data for financial rule coverage deeply. Do not assess test mechanics — that's for the QA engineers.
