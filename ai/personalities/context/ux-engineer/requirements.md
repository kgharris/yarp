# UX Engineer — Requirements Phase

## Focus

- Are user cues fully specified — will the user know what a displayed value represents, what denomination it's in, and what assumptions drive it?
- Do the requirements support the user forming a mental model of the application and its data — can they trace from inputs (assumptions) through to outputs (projections) without getting lost?
- Is the complexity of financial modeling managed — are there requirements for defaults, grouping, and layered detail so the user isn't confronted with everything at once?
- Are the relationships between views clear (what leads to what, what context carries over)?

## Artifacts

Read [requirements/ux/](../../../../requirements/ux/) exhaustively. Do not read design artifacts — if a requirement is only understandable by reference to a design document, that is a layer violation.
